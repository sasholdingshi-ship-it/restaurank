Les Algorithmes SEO Local + GEO

Comment Google classe les restaurants dans le Local Pack, et comment ChatGPT / Perplexity / Google AI choisissent quel restaurant recommander. La base technique pour construire ta webapp.

Algorithme Google Local Search — Comment ça marche

Google utilise 3 piliers fondamentaux pour classer les résultats locaux (Local Pack = les 3 résultats Maps). En 2026, les poids ont évolué avec l'importance croissante des signaux d'engagement et d'entité.

Pertinence (Relevance)

Correspondance entre la requête de l'utilisateur et le profil du restaurant. Google compare les mots-clés recherchés avec ta catégorie, ta description, tes attributs, les mots dans tes avis et ton site web.

Distance (Proximity)

Proximité physique entre l'utilisateur et le restaurant. C'est le signal le plus puissant pour les requêtes "near me" / "à proximité". Tu ne peux pas l'influencer directement — mais tu peux optimiser pour des quartiers spécifiques.

Notoriété et autorité du restaurant sur le web. En 2026, la proéminence pèse davantage sur les signaux d'engagement réel (clics, appels, itinéraires) que sur la simple quantité de backlinks.

Détail de chaque facteur — Ce que ta webapp doit auditer

📊 Google Business Profile (32%)

Critique

Le facteur n°1. 8 des 10 signaux les plus importants viennent du GBP. Ce que Google regarde :

  • → Catégorie principale — Signal n°1 absolu. "Restaurant italien" vs "Restaurant" fait toute la différence
  • → Catégories secondaires — Jusqu'à 9 catégories complémentaires (pizza, pâtes, livraison…)
  • → Nom de l'entreprise — Le mot-clé dans le nom aide (controversé mais prouvé)
  • → Description (750 car.) — Mots-clés géo + type de cuisine + spécialités
  • → Photos — Quantité, qualité, fraîcheur. Min. 20 photos récentes
  • → Attributs — Terrasse, WiFi, accessible PMR, livraison, végétarien…
  • → Horaires — Complets + horaires spéciaux (fériés). "Ouvert maintenant" = facteur top 5
  • → Google Posts — Publications régulières = signal de fraîcheur
  • → Q&A — Réponses aux questions = contenu indexable
  • → Menu — Uploadé et structuré

⭐ Avis / Reviews (20%)

CritiqueEn hausse

En forte progression (était 16% en 2023, maintenant 20%). Google évalue les avis de manière sophistiquée :

  • → Note moyenne — 4.2+ est le minimum pour apparaître dans le Local Pack
  • → Volume — Nombre total d'avis vs concurrents locaux
  • → Fraîcheur — Avis récents > avis anciens. Distribution naturelle dans le temps
  • → Contenu textuel — Les avis avec texte comptent plus que les étoiles seules
  • → Mots-clés dans les avis — "excellente pizza" dans un avis boost le ranking pour "pizza"
  • → Réponses du propriétaire — Google favorise les profils qui répondent aux avis
  • → Sentiment — Analyse NLP du sentiment global (positif/négatif/thèmes)
  • → Crédibilité du reviewer — Local Guides et reviewers actifs ont plus de poids

🌐 On-Page / Site Web (~15%)

Important

Le site web du restaurant reste important pour valider l'autorité et la pertinence :

  • → Title tag — "Restaurant Italien Paris 11 | [Nom]" — ville + cuisine dans le title
  • → NAP sur le site — Nom, adresse, téléphone identiques au GBP, crawlable
  • → Schema.org — Restaurant, Menu, AggregateRating, OpeningHours, GeoCoordinates
  • → Vitesse mobile — Core Web Vitals (LCP < 2.5s, CLS < 0.1, INP < 200ms)
  • → Contenu localisé — Pages dédiées quartier/ville si multi-sites
  • → HTTPS + Mobile-friendly — Obligatoire en 2026

📌 Citations / NAP (~11%)

ImportantEn baisse

Cohérence du NAP (Name, Address, Phone) sur tous les annuaires. La quantité compte moins qu'avant — c'est la cohérence qui prime :

  • → Consistance NAP — Même format exact partout (pas "Rue" vs "R." vs "rue")
  • → Annuaires clés — Google, Yelp, TripAdvisor, TheFork, Pages Jaunes, Apple Maps
  • → Qualité > Quantité — 20 citations cohérentes > 100 incohérentes
  • → Mentions non structurées — Articles, blogs food, guides = citations implicites

🖱️ Signaux comportementaux (~9%)

En hausseNouveau 2026

Signal en forte hausse en 2026. Google mesure comment les utilisateurs interagissent :

  • → CTR (Click-Through Rate) — % de clics quand le resto apparaît dans les résultats
  • → Appels téléphoniques — Clics sur le bouton "Appeler" depuis GBP
  • → Demandes d'itinéraire — Clics sur "Y aller" dans Maps
  • → Visites du site — Clics vers le site web depuis GBP
  • → Engagement photos — Vues des photos, ajout de photos par les clients
  • → Dwell time — Temps passé à consulter la fiche

🔗 Backlinks (~8%) + Perso (~5%)

En baisse

Backlinks : Les liens depuis des sites d'autorité (blogs food, presse locale, guides) restent utiles mais pèsent moins qu'avant. Les liens locaux (mairie, associations de quartier) sont les plus précieux.

Personnalisation : Google adapte les résultats selon l'historique de recherche et les visites passées de l'utilisateur. Non contrôlable directement.

💡 Insight clé pour ta webapp

Le GBP à lui seul représente 32% du ranking. C'est là que ta webapp peut avoir le plus d'impact immédiat : un audit approfondi du profil Google Business avec des recommandations priorisées est la feature n°1 à construire. Ensuite, les avis (20%) et les citations (11%) sont les deux leviers les plus actionnables.

Comment chaque moteur IA choisit quel restaurant citer

🟢 ChatGPT (OpenAI)

Yelp-heavyBing

Mode sans Browse : Recommande basé sur les données d'entraînement. Favorise les restaurants fréquemment mentionnés comme "meilleur" dans des sources de confiance (guides, blogs, listes "best of").

Mode Browse / Search : Recherche via Bing, puis tire de :

  • → Yelp (source n°1 pour ChatGPT)
  • → Foursquare (base de données locale)
  • → TripAdvisor (avis + classement)
  • → Site web du restaurant (contenu structuré)
  • → Articles / blogs food (listes "best of")

🟣 Perplexity

Sources diversifiéesToujours live

Toujours en mode recherche live — Perplexity ne répond jamais uniquement depuis son training. Il cite toujours ses sources.

Sources utilisées :

  • → Distribution plus équilibrée entre annuaires, sites propres, et avis
  • → Plus de poids aux reviews/social media que ChatGPT
  • → News et forums — Reddit, articles presse, blogs
  • → TripAdvisor + Yelp comme sources principales

🔵 Google AI Overviews

GBP-centric+387% restos

Intégré directement dans Google Search. AI Overviews a grimpé de +387% pour les requêtes restaurant. Il tire de :

  • → Google Business Profile (source primaire — catégorie, avis, photos)
  • → Sentiment des avis — Thèmes extraits des avis Google
  • → Site web — Contenu structuré, FAQ, menu
  • → Qualité photos — Google analyse le contenu visuel
  • → Q&A GBP — Réponses aux questions fréquentes

Note : AI Overviews n'apparaît que sur ~7% des requêtes locales — mais en croissance rapide.

🟠 Gemini (Google)

Site-firstGoogle data

Favorise les sites propres (first-party) plus que ChatGPT. Tire naturellement de tout l'écosystème Google :

  • → Site web du restaurant (source n°1 pour Gemini)
  • → Google Business Profile
  • → Google Maps data
  • → YouTube (si vidéos du restaurant existent)

Les 8 facteurs de "citabilité" par les IA

Ce qui fait qu'un restaurant sera ou ne sera pas recommandé par un LLM.

1. Présence sur les "best of" lists

Impact très élevé

Les LLM adorent les listes "Top 10 restaurants italiens à Lyon". Si le restaurant apparaît dans ces listes sur des blogs food, guides locaux, magazines — il sera cité.

2. Contenu structuré et rich

Impact très élevé

FAQ, menu détaillé avec descriptions, page "À propos" avec histoire, spécialités décrites en langage naturel. Les IA ont besoin de texte à extraire et résumer.

3. Volume & qualité des avis

Impact élevé

Les avis servent de "preuve sociale" pour les IA. Un restaurant avec 500 avis à 4.5 sera plus cité qu'un restaurant avec 20 avis à 4.8.

4. Schema.org / données structurées

Impact élevé

Le balisage schema.org (Restaurant, Menu, Review, FAQ) aide les crawlers IA à comprendre et extraire les données proprement.

5. Mentions non liées (unlinked mentions)

Impact moyen-élevé

Les mentions dans des articles, forums, Reddit, même sans lien = signaux d'autorité pour les LLM. "Le restaurant X dans le Marais est excellent" dans un post Reddit compte.

6. Complétude des profils annuaires

Impact moyen-élevé

Plus les fiches Yelp, TripAdvisor, TheFork sont complètes (photos, description, horaires, menu), plus l'IA a de données pour recommander.

7. Fraîcheur du contenu

Impact moyen

Des avis récents, un site mis à jour, des posts Google réguliers = signaux de restaurant actif et fiable.

8. Autorité du domaine

Impact moyen

Un site avec un bon DA (Domain Authority), des backlinks de qualité, et une ancienneté = plus de confiance pour les IA.

Les deux systèmes se chevauchent mais ont des différences fondamentales. Comprendre les deux permet à ta webapp d'optimiser sur les deux fronts simultanément.

Ce qui aide le SEO mais PAS le GEO

  • Proximité géographique (distance physique)
  • Signaux comportementaux (CTR, appels, itinéraires)
  • Personnalisation basée sur l'historique utilisateur
  • Google Posts (pas crawlé par les IA externes)

Ce qui aide le GEO mais PAS le SEO

  • Présence sur les listes "best of" / articles food
  • Mentions sur Reddit, forums, réseaux sociaux
  • FAQ structurée avec réponses longues et naturelles
  • Profil Yelp complet (source primaire ChatGPT)
  • Profil Foursquare (base de données IA sous-jacente)

Ce qui aide LES DEUX (synergies)

  • Google Business Profile complet et optimisé
  • Avis nombreux, récents, avec du texte riche
  • Site web avec contenu structuré et schema.org
  • NAP consistant sur tous les annuaires
  • TripAdvisor / TheFork bien remplis
  • Backlinks de sites d'autorité (guides, presse)
