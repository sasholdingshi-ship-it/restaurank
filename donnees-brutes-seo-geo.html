Facteurs Review détaillés (20% du Local Pack)

FacteurDétail précis
High Google star ratingsNote ≥ 4.2 = seuil minimum pour apparaître dans le Local Pack de façon régulière. Les tops rankent à 4.4+
Number of reviews with textLes avis avec texte comptent plus que les avis étoiles seules. Le texte est analysé pour les keywords.
Recency of reviewsWhitespark 2025 : "le facteur le plus sous-estimé". Un restaurant avec 10 avis/mois récents bat un restaurant avec 500 avis anciens.
Steady growth over timeDistribution naturelle dans le temps. Pic soudain de 50 avis = potentiel filtre spam. Croissance régulière = signal positif.
Keywords in reviews"Excellente pizza margherita" → boost pour "pizza" + "margherita". Google affiche ces mots en gras dans les "Review Justifications".
Owner responses to reviewsTaux de réponse élevé = signal d'activité. Les réponses avec des mots-clés pertinents contribuent aussi.
Diversity of review platformsAvis sur Google + TripAdvisor + TheFork = meilleur signal qu'avis uniquement Google.
Reviewer authorityAvis de Local Guides (niveau 4+) et reviewers actifs pèsent plus. Avis de comptes nouveaux pèsent moins.

Facteurs GBP détaillés (32% du Local Pack)

Signal GBPSpec technique / Ce que Google vérifie
Primary categoryListe exacte Google : "Restaurant italien", "Pizzeria", "Restaurant japonais", "Bar à sushis", "Crêperie", etc. Il y a 4 000+ catégories. La catégorie doit être la plus spécifique possible.
Additional categoriesMax 9 catégories secondaires. Ex pour un resto italien : "Pizzeria", "Livraison de repas", "Restaurant pour le déjeuner", "Traiteur".
Business titleLe nom exact légal. Si "Pizza Roma" → keyword "pizza" est intégré. Google pénalise le keyword stuffing ("Pizza Roma — Meilleur Restaurant Italien Paris 11").
Description750 caractères max. Doit contenir : type de cuisine, quartier/ville, spécialités, ambiance. Pas de keywords forcés. Langage naturel.
PhotosGoogle compte : nb photos total, nb photos récentes (< 3 mois), photos de l'extérieur (facade), intérieur, plats, équipe. Min recommandé : 25+. Les photos avec EXIF (géolocalisation) aident.
Opening hoursHeures normales + "More hours" (happy hour, brunch, dernière commande) + "Special hours" (jours fériés). Signal #5 en 2026.
AttributesGoogle offre ~80 attributs pour restaurants : Terrasse, Wi-Fi, Parking, Accessible PMR, Végétarien, Halal, Casher, Livraison, Click & Collect, Réservation, Animaux acceptés, etc.
Google PostsTypes : Update, Offer, Event. Durée de vie : 7 jours (events = jusqu'à la date). Fréquence recommandée : 1/semaine min. Inclure CTA (Réserver, Commander, En savoir plus).
Q&ALe propriétaire peut poser ET répondre à ses propres questions. Contenu indexable. Ex : "Avez-vous une terrasse ?" → "Oui, 30 places en terrasse couverte."
MenuUpload direct ou lien URL. Google a lancé le scan IA de menus photos (2025). L'API permet updateFoodMenus avec prix, descriptions, infos nutritionnelles.
Booking linkLien de réservation intégré (TheFork, OpenTable, ou site propre). "Reserve a table" button visible.
Website URLLien vers le site officiel. Le site doit avoir le NAP identique au GBP. Google vérifie la cohérence.

APIs Annuaires — Disponibilité technique

Source : Documentation officielle de chaque plateforme — Mars 2026

Google Business Profile API API disponible Read + Write

Base URL : mybusinessbusinessinformation.googleapis.com/v1

Endpoints clés :

accounts.locations.list — Lister les fiches
accounts.locations.get — Détail d'une fiche
accounts.locations.patch — Modifier une fiche
accounts.locations.getFoodMenus — Récupérer le menu
accounts.locations.updateFoodMenus — Mettre à jour le menu
accounts.locations.reviews.list — Lister les avis
accounts.locations.reviews.updateReply — Répondre à un avis
accounts.locations.media.list — Lister les photos
accounts.locations.media.create — Uploader une photo
businessProfilePerformance — Métriques (impressions, clics, appels)

Accès : Demande formelle requise. Nécessite OAuth 2.0. Quota : dépend du projet.

Limites : Pas de création de fiche ex nihilo via API (doit exister ou être claimée). Pas de modification du nom d'entreprise via API.

Yelp Fusion API API disponible Read only

Base URL : api.yelp.com/v3

/businesses/search — Recherche par nom/lieu
/businesses/{id} — Détail (nom, note, nb avis, catégories, photos, horaires, prix)
/businesses/{id}/reviews — 3 derniers avis (extrait)
/businesses/matches — Match par NAP exact

Accès : Clé API gratuite. 5 000 calls/jour gratuit.

Limites : Read-only (pas de création/modification de fiche). Max 3 avis par call. Pas de réponse aux avis via API.

Foursquare Places API API disponible Read only

Base URL : api.foursquare.com/v3/places

/search — Recherche par nom/lieu/catégorie
/{fsq_id} — Détail (nom, adresse, catégories, heures, popularité, photos, tips, prix)
/{fsq_id}/photos — Photos du lieu
/{fsq_id}/tips — Avis/tips des utilisateurs
/match — Match par NAP

Accès : Tier gratuit : 200 calls/jour. Plan payant : $200+/mois.

Limites : Read-only. Important car Foursquare alimente beaucoup d'apps IA en données de POI.

TripAdvisor Content API Accès limité Read only

Base URL : api.tripadvisor.com/api

/location/search — Recherche
/location/{id}/details — Détail (note, nb avis, classement, prix, cuisine, horaires)
/location/{id}/reviews — Avis
/location/{id}/photos — Photos

Accès : Programme partenaire requis. Pas d'accès public libre. Formulaire d'application.

Limites : Read-only. Accès restreint. Pas de modification possible des fiches.

Apple Maps Connect (MapKit) API disponible Read + Write

URL : mapsconnect.apple.com + MapKit JS/Server API

Apple Business Connect — Claim & manage listing
MapKit Server API — Search, geocode, directions
Listings : nom, catégorie, adresse, horaires, photos, showcases

Accès : Gratuit via Apple Business Connect (interface web). MapKit API nécessite Apple Developer Account ($99/an).

Limites : Gestion via interface web principalement. API de gestion de fiches limitée. Important pour Siri et Apple Intelligence.

Bing Places API disponible Read + Write

URL : bingplaces.com + Bing Maps API

Bing Places for Business — Claim & manage listing
Import depuis Google Business (sync automatique possible)
Bing Maps API — Search, geocode
Bulk upload via CSV pour multi-locations

Accès : Gratuit. Sync depuis GBP en 1 clic.

Limites : Moins utilisé en France. Crucial pour Copilot / Bing Chat (source directe).

TheFork (LaFourchette) Pas d'API publique

Accès : Aucune API publique. Gestion des fiches uniquement via TheFork Manager (interface web pro).

Workaround : Scraping possible (Puppeteer/Playwright) pour vérifier la présence et les données publiques. Pas de création automatisée.

Importance : N°1 réservation en France. Intégré dans Google. Critique pour le marché FR.

Pages Jaunes (SoLocal) Pas d'API publique

Accès : Offre "Presence Management" payante pour la gestion de fiches. Pas d'API publique.

Workaround : Scraping pour audit. Création/modification manuelle uniquement.

Importance : Poids SEO historique fort en France. Signal de citation classique.

Facebook / Instagram API disponible Read + Write

Base URL : graph.facebook.com/v19.0

/{page-id} — Infos page (adresse, horaires, catégorie, note)
/{page-id}/ratings — Avis/recommandations
/{page-id}/photos — Photos
/{page-id}/posts — Publications
POST /{page-id} — Modifier les infos

Accès : Facebook App + Page Token. Review process pour permissions avancées.

Uber Eats / Deliveroo / Just Eat APIs partenaires

Accès : APIs réservées aux partenaires intégrés (POS, agrégateurs type Deliverect). Pas d'accès public.

Workaround : Scraping des données publiques (nom, note, nb avis, lien) pour l'audit.

Importance : Les fiches Uber Eats/Deliveroo sont indexées par Google et citées par les IA. Valent d'être auditées.

OpenAI API (pour tests GEO) API disponible

Base URL : api.openai.com/v1/chat/completions

Usage : Envoyer des requêtes type "meilleur restaurant italien Paris Marais" et analyser si le restaurant client est cité dans la réponse.

Modèles : gpt-4o, gpt-4o-mini. Avec web search activé pour simuler le comportement réel de ChatGPT.

Coût : ~$2.50/1M tokens input (gpt-4o-mini). ~$10/1M tokens (gpt-4o).

Perplexity API (pour tests GEO) API disponible

Base URL : api.perplexity.ai/chat/completions

Usage : Même principe — tester les citations. Perplexity cite toujours ses sources avec URLs.

Modèles : sonar-pro, sonar. Toujours avec search activé.

Coût : ~$1/1000 requêtes (sonar). $5/1000 (sonar-pro).
